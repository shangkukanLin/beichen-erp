# 北辰 ERP 项目解读与开发指引

> 屏幕行业 ERP 全链路管理系统 | Spring Boot 3 + Vue 3 + MyBatis-Plus + Sa-Token
> 全栈可本地自建、零托管

---

## 一、项目定位与技术栈

**业务背景**：屏幕行业（LED 玻璃/排线/屏幕总成）研发→采购→委外→库存→销售→财务全链路 ERP。

**架构**：前后端分离 + 单 Spring Boot 单体（带多租户），未来可分拆微服务。

| 层级 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Spring Boot | 3.2.5 |
| JDK | Java | 21 (LTS) |
| ORM | MyBatis-Plus | 3.5.5 |
| 鉴权 | Sa-Token | 1.37.0 |
| 数据库 | MySQL | 8.0 |
| 缓存 | Redis | 7 |
| 接口文档 | SpringDoc OpenAPI | 2.3.0 |
| 工具库 | Hutool | 5.8.26 |
| 密码加密 | BCrypt (spring-security-crypto) | — |
| 前端框架 | Vue 3 + TypeScript | 3.4 / 5.x |
| 构建工具 | Vite | 5.x |
| UI 组件库 | Element Plus | 2.6 |
| 状态管理 | Pinia | 2.x |
| HTTP 客户端 | Axios | — |

**部署**：`docker-compose.yml` 一键启动 MySQL 8 + Redis 7（本地开发）。全组件目标为私有化自建，零云托管。

## 三、后端横切机制（务必先理解）

### 3.1 多租户

- `CompanyContext`：ThreadLocal 存储当前登录的公司 ID。超管（companyId=0/null）豁免所有租户过滤。
- `CompanyTenantHandler`（实现 `TenantLineHandler`）：MyBatis-Plus 多租户拦截器，自动为所有带 `company_id` 的表注入 `WHERE company_id = ?` 条件。**豁免表**：`sys_company`、`sys_user`、`sys_role`、`sys_menu`、`sys_role_menu`、`sys_user_role`、`material`、`material_bom`、`outsource_warehouse_stock`、`outsource_material_stock`、`outsource_material_bom`。
- `MetaObjectHandler`：insert 时自动填充 `companyId`（取自 `CompanyContext`）。

### 3.2 鉴权

- `SaTokenConfig`：`SaInterceptor` 拦截 `/api/**`，白名单仅 `/api/auth/login`、`/api/company/admin/verify`、`/api/company/list`、`/api/system/menu/tree/user`、swagger/doc 相关。
- 登录后 session 存入 `companyId/roles`。请求结束后清除 `CompanyContext`。
- `StpInterfaceImpl`：实现角色获取（用于 `@SaCheckRole`），**按钮级权限未实现**（`getPermissionList` 返回空）。

### 3.3 统一编码

所有业务单据自动生成 `前缀-YYYYMMDDNNN` 编码，同日序号递增（带并发重试）：

| 前缀 | 模块 | 前缀 | 模块 |
|------|------|------|------|
| CG | 采购订单 | RK | 采购入库 |
| XS | 销售订单 | CK | 销售出库 |
| PD | 盘点 | DB | 调拨 |
| QT | 其他出入库 | SK | 收款单 |
| FK | 付款单 | ZD | 账单 |
| DEV | 研发项目 | CU | 客户 |
| BUG | 缺陷 | PO/RO/DO | 外协订单/外协入库/外协发货 |

### 3.4 状态机

单据普遍采用「草稿 → 审核/确认 → 完成/作废」状态字段驱动，未引入 Flowable 工作流引擎。

### 3.5 金额精度

所有金额/数量字段 `DECIMAL(18,4)`，Service 层用 `BigDecimal` 运算，禁止浮点。

### 3.6 统一响应

`R<T>`（@Data）：`code(200/500)`、`msg`、`data`。静态方法 `ok()/ok(data)/fail(msg)/fail(code,msg)`。

## 五、业务闭环与核心流程

### 5.1 进销存 + 财务主链路（已打通自动流转）

```
供应商 ──采购订单──► 采购入库(审核) ──+库存(+流水) +生成应付台账+资金流水
                                        │
客户 ──销售订单──► 销售出库(审核) ──+库存(校验可用量+流水) +生成应收台账+资金流水
                                        │
收款单(审核)──核销应收 +写资金流水 +更新账户余额
付款单(审核)──核销应付 +写资金流水 -更新账户余额
账单生成 ──按账期/往来单位汇总未结清应收应付
```

**核心服务**：`InventoryWarehouseStockService.changeStock()`
- 按 `(warehouse_id, product_name)` 定位库存行，正数累加、负数扣减
- 扣减前校验剩余量不为负（`BusinessException("库存不足")`）
- 每次变动写 `inventory_stock_log`（前后数量、变动类型、关联单据号）

### 5.2 各模块业务逻辑概要

**采购**：`PurchaseOrderController` + `PurchaseInboundController`。采购订单审核后生成入库单；采购入库审核时调用 `stockService.stockIn()` 增加库存 + 生成 `FinancePayable`（应付台账）+ `FinanceCashflow`（资金流水）。

**销售**：`SaleOrderController` + `SaleOutboundController`。销售订单审核后生成出库单；销售出库审核时调用 `stockService.changeStock()` 减少库存（校验可用量）+ 生成 `FinanceReceivable`（应收台账）+ `FinanceCashflow`。

**库存**：`InventoryStockController`（库存查询/流水）、`StockTakeController`（盘点）、`TransferController`（调拨）、`OtherIoController`（其他出入库）。所有操作最终调用 `changeStock`。

**财务**：`FinanceReceivableController`、`FinancePayableController`、`FinanceReceiptController`、`FinancePaymentController`、`FinanceCashflowController`、`FinanceBillController`。收款/付款审核自动核销往来账 + 更新账户余额 + 写资金流水。账单按账期/往来单位汇总未结清台账。

**研发**：项目创建自动初始化 11 个固定时间线节点、按启用 BOM 类型自动建 `dev_bom` 记录。Bug/图纸/文档挂在项目下。文件上传落 `/workspace/uploads`。

**外协**：订单挂在工厂（supplier）下，含产品明细 + 物料明细（按标准用量/损耗率发料）。外协仓库库存独立（`outsource_warehouse_stock`），不走进销存库存体系。发货单支持跨仓库/供应商直发。合同模板可 HTML 配置。

## 七、前端架构详解

### 7.1 路由结构

静态路由（`createWebHistory`），`/` 为布局父路由:

```
/ ── layout/index.vue (需鉴权)
 ├── /dashboard             仪表盘
 ├── /material              物料管理
 ├── /material/bom          物料 BOM
 ├── /supplier/*            供应商（5 条路由，solusion/factory/product/material/detail）
 ├── /dev/*                 研发（project/bom/bom-type/drawing）
 ├── /outsource/*           外协（delivery/material/warehouse/order/contract-template）
 ├── /inventory/*           进销存（customer/purchase/stock/sale/warehouse）
 ├── /finance/*             财务（receivable/payable/bill/cashflow/receipt/payment）
 └── /system/*              系统（user/role/menu）
/login                       登录页
/company-manage             超管入口（免登录）
/403                        无权限页
```

### 7.2 菜单渲染

菜单**完全由后端 `sys_menu` 表驱动**，登录后 `userStore.menus` 存入 localStorage。`layout/SideMenu.vue` 递归渲染 catalog/menu 两级，根据 `icon` 字段映射 Element Plus 图标组件，`<el-menu router>` 点击跳转。

### 7.3 状态管理

`stores/user.ts`：`token/userInfo/menus` 持久化 localStorage；`isAdmin` 判断 `super_admin/admin`；`menuPaths` 收集全量菜单路由用于 403 判断；`logout()` 清除状态并跳登录。

### 7.4 HTTP 客户端

`utils/request.ts`：axios `baseURL:/api`，请求拦截自动带 `Authorization: ${token}`，响应拦截 `code===401` → 登出跳登录、`403` → `ElMessage` 报错、其他错误 `ElMessage`。

### 7.5 API 分层

`src/api/` 目录：`auth/system/material/purchase/sale/inventory/finance/customer/company/contract-template.ts`。

**注意事项**：
- 供应商接口在 `system.ts` 内（非独立 `supplier.ts`）
- 研发项目/图纸/Bug 接口在 `system.ts` 内（非 `dev/` 目录）
- 外协部分接口页面直接调 `request.get('/outsource/...')`
- 存在 8 个 `system_*.ts` Git 合并副本（`BACKUP/BASE/LOCAL/REMOTE_819/882`），正式只用 `system.ts`

### 7.6 UI 约定

Element Plus 通过 `unplugin-auto-import + unplugin-vue-components` 按需导入。**无共享 CRUD 组件**，各页面手写"筛选栏 + el-table + 分页 + 弹窗/抽屉表单 + audit/cancel"范式（约定而非复用）。

## 九、已知问题与后续开发重点

### 9.1 高优先级

1. **库存维度用 `product_name` 字符串而非 `material_id`**：应重构为 `(warehouse_id, material_id, company_id)` 唯一键 + `available_quantity`，避免同名物料错乱、支撑精确可用量校验与物料关联。
2. **`inventory_warehouse_stock` 唯一键缺 `company_id`**：`uk_warehouse_product(warehouse_id, product_name)` 应含 `company_id`。
3. **按钮级权限未启用**：`StpInterfaceImpl.getPermissionList()` 返回空，`@SaCheckPermission` 未生效。

### 9.2 中优先级

4. **文件上传路径硬编码** `/workspace/uploads`（Linux），Windows 本地运行失败，应改为可配置。
5. **无前端共享 CRUD 组件**，新模块需手写全套列表/表单/详情逻辑。
6. **噪声文件清理**：根目录 `build-*.txt`、`src/api/system_*.ts` 备份文件、`server.py`/`preview-server.js`/`contract_template_default.html` 等辅助脚本。
7. **`DEV-` 编码并发重试未统一**：仅研发项目实现了并发重试，建议提取为公共工具方法。
